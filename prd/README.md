# PRD 文档目录

本目录用于沉淀 `Claw Go` 项目的产品需求、项目现状、问题分析与后续优化建议。

## 当前已整理文档

1. `01-项目现状与产品定位.md`
   - 项目当前定位
   - 现有信息架构
   - 当前代码与架构观察

2. `02-项目优化分析与建议.md`
   - 从产品、交互、架构、稳定性、可维护性几个维度梳理优化点
   - 附优先级建议

3. `03-版本优化路线建议.md`
   - 按 P0 / P1 / P2 给出阶段性推进建议

4. `04-Agents能力补齐产品需求文档.md`
   - 基于 `/agents` 页面未授权态梳理的能力规划
   - 明确连接、Agent 资产、会话、诊断四层建设方向

5. `05-Agents能力分阶段路线图.md`
   - 按 P0 ~ P4 拆分 Agents 能力补齐节奏
   - 明确每阶段交付项、验收标准与依赖风险

## 当前结论

当前仓库已经有明确的产品方向和较完整的 Flutter 分层结构，但还缺少一套单独的 PRD 沉淀区。现已补建本目录，后续可持续在这里补充：

- 需求背景
- 页面目标
- 功能边界
- 交互说明
- 版本规划
- 风险与决策记录


6. `06-Tab重复功能梳理与收口方案.md`
   - 盘点各一级 Tab 的重复功能、重复内容与职责冲突
   - 给出导航收口与功能归位建议

7. `07-交互与UI优化规划（一级列表原则）.md`
   - 以“不出现多级列表”为核心约束梳理 UI / 交互优化方向
   - 给出一级导航、页内 Tab 与反模式清单
