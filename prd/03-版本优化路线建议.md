# 03｜版本优化路线建议

## 一、目标

本路线建议用于帮助 `Claw Go` 在不失控扩张的前提下，逐步从“已具备能力的桌面工作台”演进成“真正对普通用户友好的桌面 AI 助手”。

---

## 二、P0：先把产品边界和文档补齐

### 目标
先解决“方向清楚但边界还不够稳”的问题。

### 建议动作

1. 建立 `prd/` 文档目录并持续维护。
2. 补齐以下最小产品文档：
   - 首页定位
   - 我的任务定位
   - 设置定位
   - 高级工具定位
3. 明确主导航与高级工具的边界。
4. 明确首页、任务页、设置页的职责分工。

### 阶段产出
- 一套稳定的产品叙事
- 一套可复用的文档结构
- 一份明确的信息架构说明

---

## 三、P1：做产品层与交互层收口

### 目标
让普通用户入口更轻，让页面职责更清晰。

### 建议动作

1. 首页继续减法
   - 收敛到底层概念暴露
   - 强化“发起任务”和“继续当前任务”的主目标

2. 任务页做强化
   - 更明确区分运行中 / 已完成 / 已失败
   - 增加结果摘要和失败建议

3. 设置页做分层
   - 偏好设置与高级工具在视觉和心智上进一步拆开

4. 导航结构重构
   - 主导航保留核心页面
   - 高级工具收进二级区域

### 阶段产出
- 更易懂的主界面
- 更清晰的任务处理流程
- 更稳定的信息架构

---

## 四、P2：做底层架构治理

### 目标
让后续迭代不被核心大类拖慢。

### 建议动作

1. 拆分 `DefaultOpenClawRuntimeAdapter`
2. 梳理任务模型与状态模型边界
3. 梳理持久化策略
4. 统一导航配置与页面注册方式

### 阶段产出
- 更易维护的 Runtime 层
- 更清晰的状态流
- 更低的后续改动风险

---

## 五、建议的执行顺序

### 第一周
- 建 prd 目录
- 写项目现状与优化分析
- 开一次产品结构梳理会

### 第二周
- 定主导航与高级工具边界
- 明确首页 / 任务页 / 设置页职责
- 形成一版导航与页面职责文档

### 第三周
- 开始调整页面层级与入口文案
- 优化首页和任务页的展示逻辑

### 第四周及以后
- 开始拆 Runtime Adapter
- 梳理状态与持久化
- 持续把设计决策沉淀到 prd 文档

---

## 六、最终判断

当前项目不缺“继续开发”的能力，缺的是“在继续开发前先把产品层次和代码层次稳住”。

所以最推荐的推进方式是：

> **先收口，再扩张；先定边界，再加能力；先让产品好懂，再让能力更强。**

这条路线会比直接继续加页面、加入口、加运行能力，更适合 `Claw Go` 当前阶段。
